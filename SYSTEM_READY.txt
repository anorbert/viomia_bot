Risk/Reward Ratio (RR)
  • Mandatory minimum: RR ≥ 2.0
  • Buy: RR = (TP - Entry) / (Entry - SL)
  • Sell: RR = (Entry - TP) / (SL - Entry)
  • All test signals created meet this requirement

Signal Distribution Flow
  1. Signal received
  2. Validated (RR, symbol, ticket)
  3. Saved to Signals table
  4. TradeLog created for requesting account
  5. SignalAccount created for EACH active account
  6. Bot evaluates based on RR ratio
  7. If accepted: TradeEvent created, status = "executed"

Database Relationships
  Signals ←→ SignalAccount ←→ Accounts
      ↓
    TradeLog (trade record)
      ↓
    TradeEvent (execution record)
